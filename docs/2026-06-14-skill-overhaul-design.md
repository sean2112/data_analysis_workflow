# data-analysis-workflow Skill 全面重构设计

**日期**: 2026-06-14
**状态**: 已确认
**触发原因**: 用户发现爬取阶段 76% 数据为生成而非真实爬取，要求优化数据真实性保障

## 设计目标

1. 数据真实性：所有数据必须来源于真实爬取或用户提供的文件
2. 阶段确认制：每阶段完成后暂停，用户确认后再继续
3. 数据质量门禁：阶段间自动验证数据质量
4. 看板升级：从 plotly HTML 升级为 Streamlit 交互式看板
5. 统计集成：描述性统计、假设检验、相关性分析
6. 报告升级：数据驱动型（图表优先，文字为辅）

## Streamlit 看板设计

### 架构
```
dashboard.py
├── 侧边栏（切片器区域）
│   ├── 品牌多选器
│   ├── 价格区间滑块
│   ├── 好评率范围滑块
│   └── 自定义维度选择器
├── 主区域（图表联动）
│   ├── 行 1：KPI 指标卡
│   ├── 行 2：分布图（价格分布 + 品牌销量饼图）
│   ├── 行 3：对比图（品牌均价 + 性价比散点图）
│   └── 行 4：统计面板（描述性统计 + 假设检验 + 相关矩阵）
└── 数据表（可排序、可筛选、可下载）
```

### 切片器联动
- 所有切片器值存入 st.session_state
- 所有图表根据 session_state 过滤数据
- 切片器变化 → 页面自动刷新 → 所有图表同步更新

### 统计函数库
- 描述性统计：均值、中位数、标准差、分位数、偏度、峰度
- 假设检验：t 检验（两组）、ANOVA（多组）、卡方检验（分类）
- 相关性分析：皮尔逊、斯皮尔曼相关系数矩阵
- 用户自定义：允许用户选择列和分析方法

## 数据质量门禁

### 门禁 0（爬取后）
- 数据行数 ≥ 10
- 无完全空行
- 关键字段非空率 ≥ 80%
- 爬取数据占比 ≥ 70%（数据真实性检查）

### 门禁 1（理解后）
- 数据类型已确认
- 异常值已标注
- 缺失值策略已确定

### 门禁 2（清洗后）
- 清洗后行数 ≥ 原始行数 50%
- 缺失值为 0
- 数据类型正确
- 无重复行

### 门禁失败处理
- 展示失败原因
- AskUserQuestion 提供选择：修复后重试 / 跳过此检查 / 回退到上一阶段

## 阶段确认机制

每阶段完成后：
1. 展示阶段摘要（关键指标、产出文件、下一步建议）
2. 用 AskUserQuestion 询问用户：确认继续 / 需要修改 / 跳过后续阶段
3. 用户确认后进入质量门禁
4. 门禁通过后进入下一阶段

## 数据驱动型报告模板

### 结构
1. 执行摘要（1 页）：3-5 个 KPI 卡片 + 一句话结论
2. 关键发现（2-3 页）：每个发现 = 1 张图表 + 1-2 句解读
3. 详细分析（3-5 页）：品牌/价格/销量/统计分析，图表占 70%
4. 数据附录（1-2 页）：完整数据表 + 统计指标汇总

### 图表规范
- 中文标签
- 标题是观点陈述
- 坐标轴有单位
- 颜色统一（使用 PALETTE）
- 关键数据标注在图表上
- 图片宽度占页面 90%

## 数据真实性原则（最高优先级）

- 爬取阶段只使用实际获取到的数据
- 数据不足时必须暂停询问用户
- 绝不自行生成/编造/模拟数据
- 最终报告必须标注数据来源
- 违规示例直接列出
